## תיאור הפרויקט (Project Description)
פרויקט זה מייצג שרת RESTful API שנכתב ב-**Node.js** תוך שימוש ב-**Express** ובפרוטוקול **ES Modules (ESM)**. 
הפרויקט מנהל תיק השקעות פיננסי אישי המאפשר מעקב אחר רכישות של מניות וניירות ערך שונים בשוק ההון. השרת משלב קריאות אסינכרוניות ל-API חיצוני של **Alpha Vantage** על מנת לשלוף נתוני מסחר חיים (Real-time Stock Quotes) ולבצע רכישות מדויקות המבוססות על שערים בזמן אמת.

כל מנייה שנשמרת במערכת כוללת חישובים אוטומטיים של כמות היחידות שנרכשו לפי סכום ההשקעה וערך המנייה (מעוגל ל-3 ספרות אחרי נקודת ההצפה), הקצאת מזהה (ID) אוטומטי רציף ותיעוד זמן ההוספה בפורמט מותאם.
השרת משלב מנגנון לוגים מתקדם באמצעות **Winston Logger** המתעד כל בקשה נכנסת לשרת (HTTP requests), שגיאות אבטחה, שגיאות ריצה ומידע פנימי, ומפריד ביניהם לקבצי לוג ייעודיים.

## ארכיטקטורה ומבנה הפרויקט (Architecture & Structure)
הקוד בנוי בצורה מודולרית ומחולק לשכבות לוגיות מוגדרות היטב בהתאם לדרישות העבודה:
* **`app.js`**: קובץ האתחול הראשי של השרת, מגדיר את ה-Middlewares (CORS, Express JSON, Winston Logger) ומנתב את הבקשות לראוטר הראשי.
* **`src/routes/`**: ניתוב הבקשות (Routing) והצמדת מידלוורים של וולידציה.
* **`src/controllers/`**: ניהול בקשות ותגובות ה-HTTP, טיפול בשגיאות והחזרת סטטוסים מתאימים.
* **`src/services/`**: הלוגיקה העסקית של האפליקציה (Business Logic), כולל ביצוע הקריאות האסינכרוניות ל-API החיצוני.
* **`src/dal/`**: שכבת הגישה לנתונים (Data Access Layer) המנהלת את המערך בזיכרון המדמה מסד נתונים.
* **`src/utils/`**: פונקציות עזר ומידלוורים כלליים כגון פירמוט תאריכים, לוגר, פונקציית סינון גנרית, וולידציה של גוף הבקשה, וטיפול בטוקנים/הצפנות.

## מבנה הנתונים של השקעה במערכת (Stock Data Structure)
כל השקעה בתיק נשמרת במבנה הבא:
* `id` - מזהה ייחודי מספרי המוקצה אוטומטית (`id + 1` של המנייה האחרונה).
* `symbol` - סמל המנייה (למשל `MSFT`).
* `purchasePrice` - מחיר הקנייה ליחידה בזמן הרכישה (נלקח משדה `"05. price"` של הציטוט החי מה-API).
* `investmentAmount` - סכום ההשקעה הכולל שהוזן על ידי המשתמש.
* `shares` - מספר היחידות שנרכשו (מעוגל ל-3 ספרות אחרי נקודת ההצפה), מחושב אוטומטית לפי הנוסחה: `shares = investmentAmount / purchasePrice`.
* `addedAt` - תאריך ושעת ההוספה בפורמט ישראלי: `DD/MM/YYYY` (למשל: `22/06/2026`).

## הוראות הרצה (Setup & Running)

### דרישות קדם (Prerequisites)
* התקנת **Node.js** (גרסה 16 ומעלה מומלצת).
* מפתח API של Alpha Vantage (הוגדר מראש בקובץ `.env`).

### שלבי הרצה
1. **חילוץ והכנת התיקייה:**
   ודא כי קובץ ה-`.env` קיים בתיקייה הראשית של הפרויקט. (הערה: לצורך הגשה ב-Git וקבלת הבונוס, קובץ ה-`.env` הוכנס לקובץ `envzip.zip` כדי לא להעלותו ל-GitHub מטעמי אבטחה).
   
2. **התקנת תלויות (Dependencies):**
   פתחו טרמינל בתוך תיקיית הפרויקט והריצו:
   ```bash
   npm install
   ```

3. **הרצת השרת:**
   להרצת השרת במצב ייצור:
   ```bash
   npm start
   ```
   להרצת השרת במצב פיתוח עם ריענון אוטומטי (Nodemon):
   ```bash
   npm run dev
   ```
   השרת יופעל כברירת מחדל בפורט `5002` (או בהתאם למוגדר בקובץ `.env`) ויציג את ההודעה:
   `Server is running on port 5002`.

## נקודות קצה של ה-API (API Endpoints) & דוגמאות שימוש

השרת מאפשר פנייה ל-API תחת תחילית הנתיב `/api`. להלן פירוט הראוטים ודוגמאות לשימוש (ניתן לבצע באמצעות `cURL`, `Postman`, או הדפדפן):

### 1. שליפת כל תיק ההשקעות (כולל סינון אופציונלי)
* **נתיב:** `GET /api`
* **תיאור:** מחזיר את רשימת כל ההשקעות הקיימות בתיק.
* **סינון (Query Parameters):** ניתן להוסיף פרמטרי סינון מתקדמים:
  * `filterBy`: השדה לפיו נסנן (למשל `investmentAmount` או `purchasePrice`).
  * `condition`: תנאי הסינון (`=`, `!=`, `>`, `<`, `>=`, `<=`).
  * `value`: הערך להשוואה.
* **דוגמה לשאילתה פשוטה:**
  ```http
  GET http://localhost:5002/api
  ```
* **דוגמה לשאילתה עם סינון (מציג השקעות ששווין מעל 1000):**
  ```http
  GET http://localhost:5002/api?filterBy=investmentAmount&condition=>&value=1000
  ```
* **דוגמה לתגובה (JSON):**
  ```json
  [
    {
      "id": 1,
      "symbol": "SPX",
      "purchasePrice": 5300,
      "investmentAmount": 10600,
      "shares": 2,
      "addedAt": "12/06/2026"
    },
    {
      "id": 2,
      "symbol": "NDX",
      "purchasePrice": 18500,
      "investmentAmount": 18500,
      "shares": 1,
      "addedAt": "15/06/2026"
    }
  ]
  ```

### 2. שליפת השקעה ספציפית לפי סמל (Symbol)
* **נתיב:** `GET /api/:symbol`
* **תיאור:** מחזיר את נתוני ההשקעה עבור סמל ספציפי מתוך תיק ההשקעות המקומי (DAL).
* **דוגמה לשאילתה:**
  ```http
  GET http://localhost:5002/api/SPX
  ```
* **דוגמה לתגובה (JSON):**
  ```json
  {
    "id": 1,
    "symbol": "SPX",
    "purchasePrice": 5300,
    "investmentAmount": 10600,
    "shares": 2,
    "addedAt": "12/06/2026"
  }
  ```

### 3. שליפת נתוני מסחר חיים מ-API חיצוני (Alpha Vantage)
* **נתיב:** `GET /api/quote/:symbol`
* **תיאור:** מבצע פנייה אסינכרונית לשרתי Alpha Vantage ומחזיר ציטוט שער חי (Global Quote) עבור המניה המבוקשת.
* **דוגמה לשאילתה:**
  ```http
  GET http://localhost:5002/api/quote/MSFT
  ```
* **דוגמה לתגובה (JSON):**
  ```json
  {
    "message": "Live data fetched successfully for MSFT",
    "data": {
      "01. symbol": "MSFT",
      "02. open": "377.8200",
      "03. high": "381.3700",
      "04. low": "373.2800",
      "05. price": "379.4000",
      "06. volume": "59714157",
      "07. latest trading day": "2026-06-18",
      "08. previous close": "378.9100",
      "09. change": "0.4900",
      "10. change percent": "0.1293%"
    }
  }
  ```

### 4. רכישת מנייה על בסיס נתונים חיים
* **נתיב:** `POST /api/quote/:symbol`
* **תיאור:** מבצע פנייה אסינכרונית ל-Alpha Vantage כדי לקבל את שער המניה העדכני משדה `"05. price"`, מחשב את יחידות המניה (`shares = investmentAmount / purchasePrice` מעוגל ל-3 ספרות אחרי הנקודה), ומקצה אוטומטית מזהה רציף ייחודי ותאריך הוספה בפורמט `DD/MM/YYYY`.
* **גוף הבקשה (JSON):**
  ```json
  {
    "investmentAmount": 1000
  }
  ```
* **דוגמה לשאילתה (cURL):**
  ```bash
  curl -X POST -H "Content-Type: application/json" -d "{\"investmentAmount\":1000}" http://localhost:5002/api/quote/MSFT
  ```
* **דוגמה לתגובה (JSON):**
  ```json
  {
    "id": 4,
    "symbol": "MSFT",
    "purchasePrice": 379.4,
    "investmentAmount": 1000,
    "shares": 2.636,
    "addedAt": "22/06/2026"
  }
  ```

### 5. הוספה ידנית של השקעה
* **נתיב:** `POST /api`
* **תיאור:** מוסיף השקעה ידנית למערך (כולל חישובים אוטומטיים עבור מזהה, כמות מניות ותאריך הוספה).
* **גוף הבקשה (JSON):**
  ```json
  {
    "symbol": "AAPL",
    "purchasePrice": 175,
    "investmentAmount": 350
  }
  ```

### 6. עדכון השקעה קיימת
* **נתיב:** `PUT /api/:symbol`
* **תיאור:** מעדכן פרטי השקעה קיימת לפי סמל ומחשב מחדש את מספר היחידות במידה ומחיר או סכום ההשקעה משתנים.
* **גוף הבקשה (JSON):**
  ```json
  {
    "investmentAmount": 1500
  }
  ```

### 7. מחיקת השקעה מהתיק
* **נתיב:** `DELETE /api/:symbol`
* **תיאור:** מוחק את ההשקעה מהתיק ומחזיר את הפריט שנמחק.

## דגשים טכנולוגיים ויישום דרישות החובה בקורס

* **שימוש ב-ES Modules (ESM):** מוגדר `"type": "module"` בקובץ `package.json`. כל קבצי השרת מייבאים ומייצאים בשיטת `import/export`.
* **שימוש ב-async/await:** כל הפעולות האסינכרוניות, במיוחד קריאות ה-fetch ל-API החיצוני של Alpha Vantage, מבוצעות באמצעות תחביר `async/await` מודרני ותקין עם טיפול מלא בשגיאות (try/catch).
* **תיעוד באמצעות Logger:** שולב מידלוור מתקדם ב-`app.js` המתעד כל בקשה נכנסת לשרת לקונסול ולקבצי לוג תחת ספריית `logs/` (נוצרת אוטומטית).
* **מניעת העלאת קבצים רגישים:** קובץ ה-`.env` מוחרג לחלוטין ב-`.gitignore`. קובץ `envzip.zip` המאובטח נוצר עבור הגשת הציון במערכת המודל.
